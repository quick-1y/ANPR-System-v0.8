# Чек-лист аудита (Этап 0)

## 1) Инвентаризация текущего кода

- [ ] Карта модулей `anpr/*` с зависимостями и владельцами ответственности.
- [ ] Выделение pure business logic, которая может быть вынесена в backend без UI.
- [ ] Поиск UI-связности в пайплайнах/воркерах/инфраструктуре.
- [ ] Карта точек интеграции с внешними системами (контроллеры, RTSP, БД, файлы).

## 2) Пайплайн распознавания и производительность

- [ ] Документированы стадии: decode -> detect -> OCR -> postprocess -> persist.
- [ ] Для каждого канала описана модель конкурентности (очереди, воркеры, process pool).
- [ ] Зафиксированы узкие места CPU/GPU/IO и стоимость стадии в миллисекундах.
- [ ] Подтверждена независимая обработка каналов (изоляция отказов).

## 3) Данные и хранилища

- [ ] Зафиксирована текущая схема БД событий и индексы.
- [ ] Описаны политики хранения скриншотов и путей на диск.
- [ ] Проверены сценарии миграции схемы и обратной совместимости.
- [ ] Подготовлены требования к retention/rotation/export.

## 4) Видео и доставка в Web

- [ ] Карта типов источников (RTSP/файл/камера) и SLA по стабильности.
- [ ] Базовый sizing для WebRTC (оперативный мониторинг).
- [ ] Базовый sizing для HLS (массовый/архивный просмотр).
- [ ] Политика quality profiles: high/medium/low + динамическое переключение.

## 5) API и контракты

- [ ] Черновик REST API для CRUD каналов и управления жизненным циклом.
- [ ] Черновик API для ROI/фильтров/списков/контроллеров.
- [ ] Контракт потоковых событий (WebSocket/SSE) и форматы payload.
- [ ] Версионирование API и стратегия совместимости.

## 6) Наблюдаемость и эксплуатация

- [ ] Health-check endpoints для каждого сервиса.
- [ ] Метрики каналов: fps, latency, reconnects, timeouts, empty frames.
- [ ] Алерты и пороги деградации.
- [ ] Runbook оператора при потере каналов/росте задержки.

## 7) План вывода desktop UI

- [ ] Freeze-политика desktop ветки (только critical fixes).
- [ ] Feature parity checklist web vs desktop.
- [ ] План безопасного переключения на web UI по умолчанию.
- [ ] План удаления desktop-кода и зависимостей из CI/CD.
