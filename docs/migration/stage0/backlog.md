# Backlog миграции (черновик Этапа 0)

## Приоритет P0 (блокеры для старта Этапа 1)

1. **Выделить ANPR Core Service из desktop-кода**
   - DoR: согласована карта модулей и границы зависимости.
   - DoD: core запускается headless без импортов UI-модулей.

2. **Согласовать API-контракты управления каналами**
   - DoR: определены сущности `channel`, `roi`, `filter`, `list`.
   - DoD: описаны endpoint-ы, payload, коды ошибок, versioning.

3. **Определить базовый стек Video Gateway**
   - DoR: есть профили качества и требования по задержке.
   - DoD: выбран протокол для live (WebRTC) и для массового просмотра (HLS).

4. **Согласовать контракт событий и телеметрии**
   - DoR: утверждён список ключевых метрик.
   - DoD: описан формат event payload и health/metrics endpoints.

## Приоритет P1 (MVP web платформы)

1. Минимальный Web UI (дашборд + карточки событий).
2. Реализация WebSocket/SSE потока событий.
3. Первая версия конфигуратора каналов и ROI.
4. Нагрузочный профиль на целевое число каналов.

## Приоритет P2 (эксплуатационная зрелость)

1. Политики retention и ротации медиа.
2. Экспорт событий (CSV/JSON/API).
3. Алерты и runbook для NOC/операторов.
4. Финальное удаление desktop UI.

## Риски и меры

- **Риск:** регрессия задержек при добавлении сетевых слоёв.
  - **Мера:** budget на latency по стадиям + перф-тесты на каждом инкременте.
- **Риск:** рост стоимости транскодинга при мультиканальности.
  - **Мера:** профили качества + адаптация fps/битрейта по активности плитки.
- **Риск:** неполная замена desktop функций в web UI.
  - **Мера:** feature parity checklist и freeze desktop-функционала.
- **Риск:** деградация стабильности при reconnect storm.
  - **Мера:** backoff-стратегии, алерты и лимиты автоматических рестартов.
