# Этап 0 — Аудит и проектирование перехода на Web

Этот пакет документов фиксирует старт миграции проекта с desktop-версии на web-архитектуру.

## Цели этапа

1. Зафиксировать границы и ответственность целевых сервисов.
2. Подготовить инвентаризацию текущей реализации и зависимостей.
3. Сформировать технический backlog первой волны работ.
4. Зафиксировать риски, метрики и критерии готовности к Этапу 1.

## Состав артефактов

- `audit_checklist.md` — чек-лист технического аудита текущего состояния.
- `target_architecture.md` — целевая архитектура, контракты и НФТ-требования.
- `backlog.md` — приоритизированный backlog миграции и DoR/DoD.

## Критерии завершения Этапа 0

- [ ] Заполнен чек-лист аудита по коду, инфраструктуре, данным и эксплуатации.
- [ ] Согласованы API-контракты для управления каналами, ROI, списками и событиями.
- [ ] Зафиксированы целевые протоколы видеодоставки (WebRTC/HLS) и профили качества.
- [ ] Сформирована стратегия удаления desktop UI после достижения feature parity web-версии.
- [ ] Подготовлены метрики и алерты для контроля деградации (reconnect/timeout/latency).

## Что делаем дальше

После закрытия Этапа 0 начинается Этап 1: выделение `ANPR Core Service` и API-слоя, не зависящего от desktop UI.
