# Целевая архитектура (черновик Этапа 0)

## 1. ANPR Core Service (backend)

### Ответственность
- Декодинг и инференс на сервере.
- Пайплайн: детекция -> OCR -> постобработка -> нормализация -> событие.
- Независимая обработка каналов (один канал = отдельная очередь + lifecycle).

### Минимальные API (v1)
- `GET /api/v1/channels`
- `POST /api/v1/channels`
- `PATCH /api/v1/channels/{id}`
- `POST /api/v1/channels/{id}/start|stop|restart`
- `PUT /api/v1/channels/{id}/roi`
- `PUT /api/v1/channels/{id}/filters`
- `PUT /api/v1/channels/{id}/lists`

## 2. Video Gateway

### Ответственность
- Приём RTSP/камер.
- Доставка в web через:
  - WebRTC (минимальная задержка).
  - HLS (массовый/архивный просмотр).
- Профили транскодинга: high/medium/low.

### Правила
- Не передавать в браузер максимальное качество для всех каналов одновременно.
- Для фона снижать fps и разрешение.
- Переключать профиль по видимости/активности плитки.

## 3. Event & Telemetry Service

### Ответственность
- Поток событий распознавания в UI (WebSocket/SSE).
- Метрики и health-check.
- Алерты по reconnect/timeouts/росту задержек.

### Пример события ANPR
```json
{
  "event_id": "uuid",
  "timestamp_utc": "2026-01-15T10:12:22.144Z",
  "channel_id": "gate-1",
  "plate": "A123BC777",
  "confidence": 0.93,
  "country": "ru",
  "direction": "approaching",
  "media": {
    "frame_url": "/media/events/.../frame.jpg",
    "plate_url": "/media/events/.../plate.jpg"
  }
}
```

## 4. Web UI

### Ответственность
- Мониторинг каналов и статусов.
- Карточки событий в реальном времени.
- Конфигуратор каналов/ROI/фильтров/списков.

### Нефункциональные требования (MVP)
- Открытие dashboard <= 2 сек (без прогрева видео).
- Отображение события в UI <= 1 сек от момента сохранения.
- Деградация без падения интерфейса при потере части каналов.

## 5. Data Layer

### Ответственность
- БД событий (текущая + миграции).
- Хранилище медиа-архива (при необходимости отдельный backend/object storage).
- Retention/rotation/export.

## 6. Границы миграции

1. Сначала обеспечиваем web feature parity.
2. Затем переключаем операторов на web UI по умолчанию.
3. После этого удаляем desktop UI и связанные зависимости.
